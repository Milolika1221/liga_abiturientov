# Нагрузочное тестирование

Набор скриптов для нагрузочного тестирования сайта на 150 пользователей.

## Файлы

- `seed_users.py` - Создание тестовых пользователей в БД
- `test.py` - Основной скрипт нагрузочного тестирования
- `requirements.txt` - Зависимости Python
- `reports/` - Папка для сохранения отчетов (создается автоматически)

## Установка

```bash
cd test
pip install -r requirements.txt
```

## Использование

### 1. Создание тестовых пользователей

```bash
python seed_users.py --create --count 150
```

### 2. Запуск нагрузочного теста

**Все сценарии (по умолчанию):**
```bash
python test.py
```

**Только параллельный сценарий (150 пользователей одновременно):**
```bash
python test.py --scenario parallel --users 150 --duration 60
```

**Stress-режим (без задержек, максимальная нагрузка):**
```bash
python test.py --scenario parallel --users 150 --duration 60 --stress
```

**Только сценарий всплесков (3 всплеска по 50):**
```bash
python test.py --scenario burst --bursts 3 --burst-size 50 --pause 10
```

**Оба сценария:**
```bash
python test.py --scenario both
```

### 3. Очистка тестовых пользователей

```bash
python seed_users.py --cleanup
```

## Параметры теста

| Параметр | Описание | По умолчанию |
|----------|----------|--------------|
| `--scenario` | Тип сценария: parallel, burst, both | both |
| `--users` | Общее количество пользователей | 150 |
| `--concurrent` | Одновременно активных пользователей | **150** |
| `--duration` | Длительность теста (сек) | 60 |
| `--bursts` | Количество всплесков | **3** |
| `--burst-size` | Размер всплеска | 50 |
| `--pause` | Пауза между всплесками (сек) | 10 |
| `--stress` | Режим без задержек (максимальная нагрузка) | - |

## Сценарии

### Параллельный (parallel)
Все 150 пользователей работают **постоянно и одновременно** на протяжении всего теста. Каждый пользователь циклически выполняет сценарий: логин → профиль → документы → загрузка достижения → таблица лидеров → категории.

- Задержки между запросами: 0.05-0.1 сек (минимальные)
- Задержка между циклами: 0.1-0.3 сек
- **Stress-режим (`--stress`)**: полностью отключает все задержки для максимальной нагрузки

### Всплески (burst)
Группы пользователей (по 50) выполняют действия одновременно с паузами между всплесками. Имитация пиковой нагрузки с простоями.
- 3 всплеска × 50 пользователей = 150 пользователей

## Отчеты

Отчеты сохраняются в папке `reports/` в двух форматах:
- `.json` - Полные данные для анализа
- `.txt` - Читаемый текстовый отчет

## Проверяемые endpoints

- POST /login - Авторизация
- GET /profile/:id - Получение профиля
- GET /user-documents/:userId - Получение документов
- POST /documents/upload - Загрузка достижения (30% пользователей)
- GET /leaderboard - Таблица лидеров
- GET /categories - Категории мероприятий

## Метрики

- Общее количество запросов
- Успешность запросов (%)
- Среднее/мин/макс время ответа
- P50, P95, P99 времени ответа
- Запросов в секунду (RPS)
- Статистика по каждому endpoint
- Список ошибок
